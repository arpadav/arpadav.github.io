    gfs

    June 11, 2026 18:25:02 -0400

    graph based file system. no not mergefs or others. ive wanted this for data curation and handling for a while, where many scientists + engineers will default to FS for experimentation and training, then ad-hoc scripts everytime they want to curate a dataset. main idea is something like redis/duckdb with vector embeddings or just neo4j instance which can be mounted using POSIX. therefore, deletion of a file = deletion of a node, so it gets removed from multiple locations (this is a feature). mounting would solely be a query, which then gets compounded. 3 types of folder types: filter, node, and POSIX. filter means its an additional search filter which gets added to all child nodes. node is the last filter, which then gets executed and files appear. POSIX is a standard folder. nodes can exist in one another, filters can compound (e.g. /mnt/images/cat/outside/May-2026/ where images + cat + outside are filters (no images appear) and May-2026 is the node which gets executed as intersection of all those tags.
