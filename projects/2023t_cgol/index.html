    Conway's Game of Life using Rust + WASM

      Quick implementation of CGOL to learn WASM

      Individual Project — Status: Complete

      Nov 2023; Nov 2024 — Columbia, Maryland

    Repositories

       — rust-wasm-cgol

    (Nov 26, 2024) Overview:

    I am adding this to my website now: previously I didn't add it to my website because it was in an incomplete/shoddy state. Meaning, the drawing functionality was lacking, it wasn't as fast as it could have been, and more. Around ~2 days of development time in my free time, I revisited this to make improvements.

    The main improvements are:

    • Rayon for parallelization
    • Other speed improvements (WASM optimization flags, general algorithm changes, different way to render images on screen, etc.)
    • Aesthetic improvements (user can select colors, select options, etc.)
    • Ability to import/export states (because why not, its fun)

    (Nov 6, 2023) Overview:

    I am writing this in 2024, so I can't relay my exact 2023 thoughts. However, I was trying to learn web-assembly so what better way to do that than to implement Conways Game of Life! The implementation is simple, but lacking in features. Took me about 1 day to implement this and get it working.

    Conways Game of Life

    Good things:

    • Click to draw + erase (toggle cells)
    • Is resizable!
    • Can import/export game states!
    • Is very fast! Try reducing the cell size + increasing the window size

    Not-so-good things:

    • Mobile loading of initial `glider.bin` is inconsistent. Not worth the effort to debug.
    • Resizing and changing cell size will result in a new board/game/map, randomly filled!! This is intentional, but would be nice if there were alternatives. Also not worth the effort
