    2026 - One-time downloads

      NAS / local download link generation and management

      Individual Project - Status: Maintainer

      Mar 2026

    Repositories

       - otd

    Overview

    ive had a NAS for a while. as expected, you keep a majority of your files, images, etc there. ive already talked about in my notes how to set up immich proxy on truenas scale, which is nice for image and album sharing.

    but for file sharing, thats different. one, i dont want people to access my raw file system for obvious reasons, and two, i didnt want to use google drive or email it everytime. and i was looking at solutions online (maybe i didnt look hard enough?) but i couldnt find something that fit my needs:

    1. select files to share
    2. select zip/tar/archive format
    3. get shareable link
    4. manage link - N times downloaded, expires after T time

    i did see some built-in capabilities like https://www.synology.com/en-us/dsm/feature/file_sharing or everyone telling you to just make a seperate user and make a SMB share or something. none of this is what i wanted.

    so using the same mindset as the immich-public proxy and using cloudflare, it becomes stupidly easy.

    i made otd (One-time downloads) as a simple side project and proof of concept, but i just kept adding more features as time went on. now its somewhat stable and able to be released to the public, but to say its 100% complete would be lying.. still a couple of quality-of-life fixes that would for sure benefit those who would use it.

    dev stack

    rust for the file server, svelte for the dev dashboard. the dev dashboard is hosted on one port, default to loopback localhost, whereas the download server is on a seperate port all-together, default exposed to 0.0.0.0. the download server simply serves files with 0 front end, and the admin panel you can select and manage links.

    ideally you can use a cloudflare tunnel to expose the download port, or some other reverse proxy. eventually, i will upload this to the docker hub and make it available on truenas. that should hopefully be done within the month
